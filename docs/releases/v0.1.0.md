# v0.1.0 — Initial public release

Async, version-aware Python client for the **FortiManager** JSON-RPC API.
Built on `httpx` and Pydantic v2, with strict typing throughout.

## Why this exists

Fortinet renames endpoints and payload fields between releases — for
example, FMG 7.6 moved meta-variable storage from `obj/dynamic/variable`
to `obj/fmg/variable` and renamed `local-value` to `value` in
per-device mappings. A client that does not absorb those differences
silently corrupts state on upgrade.

This package isolates per-version differences in **adapter classes**
verified against golden fixtures captured from real FMGs, so callers
do not branch on version themselves.

## Supported FMG versions

| Version | Adapter | Notes |
|---|---|---|
| 7.2 | `FMG72Adapter` | Inherits the 7.4 shape (no probed divergence yet) |
| 7.4 | `FMG74Adapter` | Default shape |
| 7.6 | `FMG76Adapter` | Meta-variable namespace move, `value` rename, `os_type=fos` on model-device |

`detect_version()` reads `/sys/status` and picks the right adapter
automatically.

## What's in this release

### Transport core

- `FMGClient` / `FMGClientProtocol` — async JSON-RPC with retries
  (jittered exponential backoff) and typed error mapping.
- `SessionManager` — Bearer (API token) and user/password auth.
- `WorkspaceLockContext` / `workspace_session()` — ADOM workspace
  lock with safe `-9` degrade when workspace mode is off.
- `TaskTracker` — polls long-running FMG tasks (installs, scripts) to
  completion.

### Version awareness

- `VersionAdapter` ABC + `BaseAdapter` defaults.
- Per-version adapters: `FMG72Adapter`, `FMG74Adapter`, `FMG76Adapter`.
- `@AdapterRegistry.register("7.x")` + `detect_version()` factory.

### Domain managers

All require an explicit `adapter` keyword — no silent defaults:

`DeviceManager`, `MetafieldManager`, `InstallManager`,
`CLITemplateManager`, `CLITemplateGroupManager`, `AddressManager`,
`AddressGroupManager`, `ServiceManager`, `ServiceGroupManager`,
`PolicyPackageManager`, `BlueprintManager`,
`ProvisioningTemplateManager`.

### Typed errors

`FMGError`, `AuthError`, `LockError`, `NotFoundError`,
`DuplicateError`, `TaskTimeoutError`, `VersionError` — each mapped
from a well-known FMG status code.

### Tests

140 unit tests covering client, session, locking, models, registry,
detection, every manager, payload shape per version, and golden
fixtures captured from a live FMG 7.6.5.
